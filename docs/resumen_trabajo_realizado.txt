3. CORRECCIÓN DEL BUG EN desacoplar_patrones
──────────────────────────────────────────────────────

La función desacoplar_patrones en el notebook tenía un bug: reemplazaba la línea
COMPLETA por [SECCION] cuando la línea caía dentro de la zona de encabezados y
además contenía el patrón. Eso producía 307 fragmentos de una sola palabra
([SECCION]) en el consolidado de Mateo.

La regla correcta según el enunciado (A3) es: encontrar la PRIMERA ocurrencia del
patrón en todo el texto y reemplazar SOLO el match, nada más.

La función corregida:

    def desacoplar_patrones(texto: str, etiqueta: str) -> tuple:
        if etiqueta not in _HEADING_COMPILADOS:
            return texto, False
        for patron in _HEADING_COMPILADOS[etiqueta]:
            m = patron.search(texto)
            if m:
                nuevo = texto[:m.start()] + TOKEN_NEUTRO + texto[m.end():]
                return nuevo, True
        return texto, False

Los 307 fragmentos dañados (LIM=249, CONC=58) quedaron corregidos en los archivos
del punto 2.


──────────────────────────────────────────────────────
4. POS_RELATIVA EN LOS DATASETS DE LIM/CONC
──────────────────────────────────────────────────────

Los dos archivos de LIM/CONC necesitaban la columna pos_rel (posición relativa del
fragmento dentro del documento fuente, de 0 a 1). Esta columna es un feature para
el encoder de Tarea 2.

Fuente: columna relative_start de los parquets
  - data/processed/task1/camilo_dataset_A.parquet
  - data/processed/task1/camilo_dataset_B.parquet
  - data/processed/task1/camilo_dataset_A_oldtime.parquet
  - data/processed/task1/camilo_dataset_B_oldtime.parquet

Para los 3549 fragmentos TRAIN_TEST: ya tenían pos_rel del proceso de etiquetado.
Para los 384 fragmentos EVAL:
  - 293 encontrados en los parquets por doc_id → valor exacto de relative_start.
  - 42 encontrados en los parquets oldtime → valor exacto.
  - 49 no encontrados en ningún parquet → fallback: media por etiqueta
    (LIM=0.394, CONC=0.524).

El dataset_desacoplado tiene la misma pos_rel que el acoplado (orden de filas
verificado idéntico, mismo origen).


──────────────────────────────────────────────────────
5. CORRECCIÓN DEL DATASET TAREA 2 (consolidado_tarea_2.csv)
──────────────────────────────────────────────────────

El dataset tenía dos problemas en POS_RELATIVA:

1. Las 416 filas de LIM y CONC tenían valores discretos como 1, 0.5, 0.333...
   que parecen calculados como (número de sección / total de secciones), no como
   posición real del fragmento. Los reemplacé con relative_start de los parquets.

2. Una fila de CONTR (doc_id=335504) tenía POS_RELATIVA='75005439.txt' —
   un nombre de archivo en lugar de un número. La corregí con la media de CONTR.

La columna quedó como float64, 0 NaN, en las 3125 filas. También eliminé la
columna Unnamed: 0 que era un artefacto de index al guardar.

8. ESTRUCTURA DE DESPLIEGUE
──────────────────────────────────────────────────────

Sergio creó el mock del demostrador en la rama streamlit-demo-mock. Su trabajo no
se toca. La arquitectura que implementé es:

  demo/         (Sergio — intacto)
    app.py
    mock_backend.py
    sample_inputs.py
    README.md

  api/          (nuestro backend real)
    main.py           FastAPI con endpoints /analyze, /compare, /models, /health
    inference.py      Lógica de inferencia: Gemini, encoder, Ollama open-weight
    model_loader.py   Carga lazy de artefactos HuggingFace desde models/
    schemas.py        Modelos Pydantic que respetan el contrato JSON del mock
    requirements.txt

  demo/
    backend_production.py   Proxy HTTP a la API — se copia sobre mock_backend.py
                            al hacer deploy (cp demo/backend_production.py demo/mock_backend.py)

  models/
    task1_encoder/    artefacto del mejor encoder T1 (pendiente: elegir modelo)
    task2_encoder/    artefacto del mejor encoder T2 (pendiente: elegir modelo)

  docker/
    Dockerfile            empaqueta la API en un contenedor Python 3.11-slim
    docker-compose.yml    levanta API (puerto 8000) + demo Streamlit (puerto 8501)

  .env.example            template de variables de entorno
  .streamlit/secrets.toml template para Streamlit Cloud


Para activar el backend real localmente:

    cp .env.example .env
    # llenar GOOGLE_API_KEY en .env
    pip install -r api/requirements.txt
    uvicorn api.main:app --reload

Para activar el Streamlit con backend real en lugar del mock:

    export BACKEND_URL=http://localhost:8000
    cp demo/backend_production.py demo/mock_backend.py
    streamlit run demo/app.py

Para levantar todo con Docker:

    docker-compose -f docker/docker-compose.yml up --build
